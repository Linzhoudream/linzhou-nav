# 常识检查点设计

## 什么时候需要检查？

在以下3种场景中，**必须**插入常识检查点：

1. **AI给出行动方案** — 尤其涉及物理操作、时间安排、人员协调时
2. **AI生成交付物** — 文档、代码、客户沟通等
3. **AI做出了判断** — 选择了某个选项而非另一个

## 检查什么？

### 三问速检法（30秒完成）

1. **物理常识**：这个方案在物理世界中行得通吗？（→具身常识层）
2. **优先级**：AI关注的主要因素，真的是最重要的吗？（→情境权重层）
3. **没人说但很重要**：有没有"所有人都知道所以没人提"的前提被忽略了？（→社会规则层）

### 红灯信号（出现任何一条就暂停）

- 方案让你本能觉得"别扭"
- 方案忽略了显而易见的物理约束
- 方案把次要因素当成了主要因素
- 方案的前提是"所有人都知道X"但AI不一定知道

## 如何设计制度化检查点？

不是每次都靠直觉，而是在流程中预设检查位置：

```
AI生成方案 → 【常识检查点】 → 人工确认 → 执行
                ↑
        专门检查隐性知识四层中
        AI可能缺失的约束
```

## 与其他工具的配合

- 检查发现问题 → 写入[[AI盲区清单]]，持续积累
- 问题根因 → 对照[[隐性知识四层模型]]，定位是哪层
- 紧急情况 → 用[[5分钟止血自查清单]]快速处理
