# 核心决策框架（通用版）

核心决策框架（通用版）【脱敏公开版 | 更新日期：2026-05-27】本内容为个人经验总结，仅供交流参考，不构成法律、合规、财务等专业建议。一、核心策略：三层递进第一层：扬长避短 → 保底线第二层：AI放大 → 提效率第三层：合作杠杆 → 拓上限1.1 扬长避短（所有选择的底线）四道筛子（必须全部通过）：筛子问题通过标准筛子1这个事精力吃得消吗？能在有限时间内完成筛子2这个事需要频繁社交吗？不需要/可通过文字异步解决筛子3这个事需要高强度抗压吗？不需要筛子4这个事需要长时间连续操作吗？不需要，可拆成短时间单元结论：任何选择，只要有一道筛子不通过，就不做。1.2 AI放大长板长板AI怎么放大系统化思考你写规则，AI生成内容文字沟通你写文字，AI生成视频/图片本地知识你提供经验，AI整理成模板AI对话能力你搭智能体，服务更多人关键：长板是"定义问题+设计规则"，AI帮你"执行+放大"。1.3 合作杠杆核心逻辑：用你的长板（脑子）换别人的长板（腿/嘴）你的角色合作者的角色设计系统、写SOP、搭智能体执行体力活、面对面沟通提供知识产品、方法论对接客户、收款开票做决策判断、避坑指导收集信息、跑腿调研二、资源优先级优先级资源类型使用原则P0知识产权（SOP、模板、方法论）优先产品化，一次制作无限复用P0信息差知识（本地避坑、行业合规）产品化变现P1信任关系（熟人认可）被动激活，有人来问才用P2决策判断力（帮人避坑、筛选方向）先用自己验证，再做咨询P2同理心陪伴服务形态，体力消耗大P3撮合对接需要社交，暂不适合三、核心定位低体力数字资产架构师核心：帮助别人把"混乱的经验"变成"有序的资产"不可替代性：本地知识 + 生存智慧 + 真人温度 + 低体力方法论四、验证标准验证失败信号信号含义下一步用户说"挺好的"但不付费礼貌性认可换问法："愿意付多少"用户说"等我有空再看"不是刚需换更痛的点拿到产品后不再联系没有复用价值补充后续服务验证成功信号信号含义用户主动追问"还有吗"真实需求被激活用户主动推荐给朋友信任传递用户愿意预付价值认可15分钟验证法遇到新想法，用15分钟三问快速判断：这事能解决真问题吗？（5分钟）这事能低成本试吗？（5分钟）这事能落地吗？（5分钟）3个"是"→立刻做；2个→先做第一个"是"的；1个或0个→放弃或暂存。五、执行检查清单每次做选择时：□ 四道筛子过了吗？□ 用的是P0/P1资源吗？□ 能产品化吗？□ 验证方法对吗？□ 有退出方案吗？每次失败后，提取3条：□ 场景是什么？□ 我的错误假设是什么？□ 正确认知是什么？六、一句话总结扬长避短保底线 + AI放大提效率 + 合作杠杆拓上限核心定位：帮人把混乱的经验变成有序的资产核心约束：精力有限、低体力、不社交——这是底线，不可突破
