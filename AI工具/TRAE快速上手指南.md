## 一、内容选题规划模板

### 通用提示词框架

```text
你是一个资深内容运营。请基于以下信息，为我规划未来一段时间的内容选题方向。

账号定位：
目标平台：
目标人群：
内容形式：
更新频率：
近期热点或行业动态：
已有内容表现：

请输出：
1. 选题方向建议（至少 5 个方向）
2. 每个方向下 3 个具体选题
3. 每个选题的推荐标题
4. 内容形式建议
5. 目标人群匹配度分析
6. 预期互动效果
7. 内容排期建议
```

### 知乎场景适配版

```text
你是一个知乎内容运营。请帮我挖掘"温区问题"——即赞数在20-200之间、回答数在5-30之间、有讨论空间但竞争不激烈的问题。

账号定位：（你的定位）
目标平台：知乎
目标人群：（你的目标读者）
内容形式：知乎回答（800-1500字）
更新频率：每周2-3篇
已有内容表现：踩坑类内容互动较高；经验复盘类内容收藏率高

请输出：
1. 本周可挖掘的3-5个温区问题（附问题链接）
2. 每个问题的当前赞数/回答数/热度趋势
3. 每个问题能否写出差异化回答（用真实经历）
4. 推荐回答结构（痛点钩子→经历→方法论→金句收尾）
5. 预期互动效果评估
```

## 三、社区运营三场景

### 场景一：内容生产与批量出稿

**核心动作：把调性沉淀成 Rules，把高频任务封装成 Skill**

> "不会直接拿它的结果发布，而是把它当成'第一轮选题会纪要'。会删掉太顺滑的句子，补上真实案例和自己的判断。这样改完的内容，才不会有那种一眼看出来的AI味。"

**批量出稿提示词：**

```text
基于这3个选题，分别产出知乎回答初稿。每篇要求：
1. 给3个标题候选：悬念型、数字型、共鸣型各一
2. 正文按"痛点钩子→踩坑经历→方法论提炼→自嘲收尾"四段式
3. 配套1段开头50字的"金句钩子"
4. 严格遵守写作规则
```

**"让它反驳你"的方法：**

```text
现在请你扮演一个挑剔的知乎老用户，针对这篇回答：
1. 找出3处"一看就是AI写的"表达并改掉
2. 指出1个"没有真实经历写不出来"的薄弱点
3. 检查有没有违反知乎AIGC政策的风险
```

### 场景三：数据复盘与用户洞察

**核心方法：让AI先整理数据→找反常识点→让它反驳你的结论→做成看板**

```text
把本周知乎回答数据发给你（阅读/点赞/评论/收藏），请你：
1. 汇总到一个统一表格
2. 做交叉分析：哪类选题互动最高？哪些时段发布效果更好？
3. 找出2个反常识的数据点
4. 验证假设："踩坑类内容确实比方法论类互动高"是否成立
```

## 四、使用建议

| 建议 | 说明 |
|------|------|
| 明确需求任务的目标 | 先教AI你的判断标准 |
| 提供更多业务上下文 | 往里塞真实经历和判断 |
| 要求给到能落地执行的结果 | 定期问它"这轮你有没有发现新东西" |

### 给AI写内容草稿时，前置补充信息模板

```text
【目标】帮我写一篇知乎回答草稿
【账号定位】（你的定位）
【真实经历】（这里填具体经历，如某工具3天17次测试）
【目标问题】（这里填知乎问题标题）
【要求】
- 用第一人称"我"
- 包含至少1个具体数字/时间/地点
- 开头用痛点钩子
- 中间有真实踩坑细节
- 结尾有自嘲或金句
- 删除"赋能/闭环/沉淀"等AI味词汇
- 总字数800-1500字
```

## 五、写 Skill 的关键原则

### 什么样的任务值得沉淀成 Skill？

| 判断维度 | 适合Skill化 | 暂时别做 |
|---------|------------|---------|
| 频率 | 每周都有人做 | 一次性脑暴 |
| 稳定性 | 流程大体固定，只是输入不同 | 每次都要重新定义目标 |
| 交付物 | 能落到文档、表格、报告 | 只是开放式讨论 |
| 风险 | 可以设置检查点和人工确认 | 高风险动作很多 |

### 三种 Skill 拆法

| 拆法 | 适合什么 | 重点写什么 |
|-----|---------|----------|
| **角色型** | 负责某一步判断（如需求分析师、内容编辑） | 判断框架、输入边界、输出标准 |
| **工具型** | 负责把工具用对（如操作飞书、处理表格） | 命令、参数、权限、异常处理 |
| **编排型** | 串起多个角色和工具（如需求→设计→任务拆解） | 执行顺序、检查点、回退路径 |

### 写 Skill 的核心原则

**别只写原则，要写动作：**

| 不要只写 | 写成这样 |
|---------|---------|
| 保证内容高质量 | 先提炼读者、目标和主线；连续纯文本超过3段时改成表格或列表 |
| 删掉AI味 | 删除"赋能/闭环/沉淀"；把"首先其次最后"改成直接说 |
| 用口语 | 每篇至少1处口语化点缀，如"中不中""得嘞""就是这个理儿" |

### 验收清单
- [ ] 用户换一种说法时，description仍能触发
- [ ] 缺少必要输入时，知道该追问、假设还是停止
- [ ] 每个步骤都有明确动作，而不只是原则
- [ ] 输出格式固定，能被下一个流程继续使用
- [ ] 至少有3个标准样例、2个边界样例、1个反例

## 六、规则与记忆系统

### 一条好规则的四个要素
1. **具体动作**，不是原则性描述
2. **有判断条件**，什么时候用、什么时候不用
3. **有输出标准**，做完长什么样
4. **有停止条件**，什么情况该停下来问人

### 内容创作规则示例

```markdown
## 内容写作规则
- 开头30字内必须有痛点钩子或冲突
- 每篇至少1个"只有真实经历才能写出来"的细节
- 连续纯文字超过3段，必须改成表格/列表/对比
- 初稿完成后，必须执行一轮"反AIGC检查"
- 总字数800-1500字，不超过2000字
```

## 七、可自动化的场景

| 任务 | 频率 | 产出 |
|------|------|------|
| 温区问题扫描 | 每周一 | 3-5个候选问题+评估 |
| 内容数据复盘 | 每周日 | 本周数据汇总+下周建议 |
| 领域热点追踪 | 每日 | 3-5条+与选题支柱匹配度 |

### 注意事项
- 先手动跑通一次流程，确认输出稳定，再自动化
- 自动化的边界要写清楚：哪些直接生成、哪些标记待确认、哪些必须人工介入
